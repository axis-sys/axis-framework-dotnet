# Bundled canonical sources

This plugin ships a verbatim copy of the framework's canonical sources so every reference resolves
offline, from the installed plugin alone:

| In this bundle | Mirrors (framework repo) |
|---|---|
| `rules/` | `rules/` — the canonical rule base (one YAML per rule) |
| `docs/` | `docs/` — the navigable documentation (en-us + pt-br) |
| `scaffolds/` | `src/scaffolds/` — the compiled, tested reference application |

Anchors inside rule files (`reference.at`, `examples.correct`) that cite `src/scaffolds/...`
resolve against `scaffolds/...` here. Anchors into the framework's own source or tests
(`src/0-Foundations/...`, `src/2-ApplicationFlow/...`, ...) are **not** bundled — they resolve only
in the framework repository itself. The scaffold copy is reference-only: it omits `bin/`/`obj/`
and the framework projects it builds against, so it is not buildable in place.

Generated by `skills/scripts/bundle-plugin.mjs`; CI fails when this bundle drifts from the sources
(`bundle-plugin.mjs --check`). Do not edit by hand.
